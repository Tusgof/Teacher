Data architecture และ schema ที่ต้องมี

ถ้าข้อมูลไม่ถูก ระบบเทรดจะดูดีใน backtest และพังใน live

Data layers

Market bars

SPY 1-min หรือ tick bars, OHLCV, timezone

Option chain

bid/ask/mid, strikes, expiration, type, quote time

Derived state

ORB high/low, VIX, realized vol, gap bucket, spread width

Orders/fills

submitted, accepted, partial fill, rejected, canceled, fill price

Portfolio

cash, margin, exposure, open risk, daily PNL

Research labels

TP first, SL first, close PNL, net PNL, no-trade reason

Timestamp discipline

ทุก row ต้องตอบได้ว่า “ข้อมูลนี้รู้ได้เมื่อไร” ไม่ใช่แค่ “ข้อมูลนี้เป็นของวันนั้น” 0DTE ใช้เวลาระดับนาที การปนข้อมูล 10:00 ไปใช้ตัดสิน 9:36 คือ leakage.

Minimal tables

underlying_barssymbol,timestampสร้าง ORB และ realized features

signalstrade_idเก็บ decision snapshot
ordersorder_idจำลอง/ตรวจ execution

risk_statetimestampdaily loss, exposure, stop trading flag

Retrieval

  1. ทำไม option quote ต้องมี bid/ask ไม่ใช่ mid อย่างเดียว
  2. decision timestamp ต่างจาก bar timestamp อย่างไร
  3. order table กับ fill table ต่างกันอย่างไร
