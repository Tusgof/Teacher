Live infrastructure และ paper trading

เปลี่ยนจาก research เป็น operation โดยไม่เอาเงินจริงไปทดสอบระบบที่ยังไม่พร้อม

Infrastructure components

Componentหน้าที่
Data feedreal-time quote, option chain, event calendar
Signal engineคำนวณ filter และ entry rule
Risk engineตรวจ size, daily limit, kill switch
Order managerส่ง, cancel, replace, handle rejection
Position monitorTP/SL/time stop, stale data, partial fill
Loggerบันทึก signal, orders, fills, decisions, errors
Dashboard/alertsให้มนุษย์เห็นความผิดปกติทันที

Paper trading gate

Live Trading Infrastructure page เตือนว่า live มี connectivity failure, order rejection, partial fills, slippage, broker constraints และ monitoring gaps. ดังนั้น paper trading ต้องทดสอบ workflow ไม่ใช่แค่ดู PNL.

Paper trading pass criteria

  • signal ตรงกับ backtest logic
  • orders ส่งตาม limit rules
  • all no-trade rules ทำงาน
  • TP/SL/time stop ทำงาน
  • logs ครบพอ audit
  • ไม่มี manual override นอกแผน
  • paper fills ถูกเปรียบเทียบกับ realistic bid/ask

Retrieval

  1. paper trading ทดสอบอะไรนอกจาก PNL
  2. risk engine ต้องมาก่อน order manager เพราะอะไร
  3. partial fill สร้างปัญหาอะไรกับ vertical spread
