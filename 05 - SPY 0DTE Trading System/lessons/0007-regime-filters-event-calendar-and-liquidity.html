Regime filters, event calendar, liquidity

เรียนวิธีลดวันที่ระบบไม่ควรเสี่ยง โดยไม่ overfit จนเหลือแต่วันที่สวยในอดีต

Filter families

Filterตัวอย่างความเสี่ยงถ้าใช้ผิด
Calendarweekday, expiration scheduleเลือกจากอดีตจน overfit
VolatilityVIX 15-25, realized volregime เปลี่ยนทำให้ threshold เก่าใช้ไม่ได้
EventFOMC, CPI, NFP, Fed speechevent time/source ไม่ตรง ทำให้ leakage
Liquiditybid/ask width, volume, quote ageมองข้าม stale quotes
Microstructuregamma/vol attenuation proxyข้อมูลอาจหาไม่ได้หรือ noise สูง

หลักจาก wiki

Regime Filtering page ย้ำว่า filter มีประโยชน์แต่เป็น overfitting risk โดยเฉพาะเมื่อ filtered trades เหลือน้อย ต้องรายงาน confidence interval, subperiod behavior และ parameter sensitivity.

กฎขั้นต่ำ

  • ทุก filter ต้องใช้ข้อมูลที่รู้ก่อน entry
  • ระบุ source ของ event calendar และเวลา update
  • ทดสอบ threshold รอบ ๆ ค่า เช่น VIX 14-24, 15-25, 16-26
  • ห้ามเพิ่ม filter หลังเห็นผล test โดยไม่เริ่ม validation ใหม่

Retrieval

  1. filter ช่วยอะไรและเสี่ยงอะไร
  2. ทำไม event calendar เป็น leakage source ได้
  3. parameter sensitivity ต้องตอบคำถามอะไร
