Walk-forward, OOS, overfitting control

ทำให้ผลวิจัยยากขึ้นแต่เชื่อถือได้ขึ้น

หลัก validation

Backtest Validation Protocol บอกให้ใช้ chronological splits, keep final test untouched, verify timestamp, include costs, test parameter sensitivity, report subperiods, label evidence type และ treat metrics as estimates with uncertainty.

Walk-forward pattern

  1. เลือก train window เช่น 252 trading days
  2. fit rule/model เฉพาะข้อมูลอดีต
  3. trade/validate ช่วงถัดไป
  4. เลื่อน window หรือ expand
  5. รวม OOS trades ทั้งหมด
  6. ห้ามกลับไปแก้ rule จาก final test โดยไม่ reset protocol

Overfitting signs

  • ผลดีเฉพาะ parameter ค่าเดียว
  • ผลดีเฉพาะ subperiod เดียว
  • feature จำนวนมากเทียบกับจำนวน trades น้อย
  • model complex แต่ไม่ชนะ simple baseline หลัง cost
  • win rate สูงแต่ payoff/tail แย่

Evidence label

ทุกผลลัพธ์ควรติดป้าย: in-sample, validation, OOS, paper trading, live-forward. ถ้าไม่มีป้าย ห้ามใช้ตัดสินเพิ่ม size.

Retrieval

  1. ทำไม random split ไม่เหมาะกับ time series
  2. parameter sensitivity บอกอะไร
  3. final test ถูกใช้ทำอะไรเท่านั้น
