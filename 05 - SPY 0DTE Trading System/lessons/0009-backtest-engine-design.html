Backtest engine: event-driven ไม่ใช่ spreadsheet ง่าย ๆ

เรียน backtest ที่ใกล้ live พอสำหรับ 0DTE ไม่ใช่คำนวณ payoff ท้ายวันแบบลอย ๆ

ทำไม event-driven

Wiki ระบุว่า event-driven backtesting จำลอง market, signal, order, fill, rebalance, cost และ portfolio events. สำหรับ 0DTE สำคัญมากเพราะ order may not fill, TP/SL sequence matters, bid/ask changes fast, และ time stop ต้องทำงานจริง.

Event loop

  1. Load market event
  2. Update indicators/filter state
  3. Generate signal if allowed
  4. Create order with limit and size
  5. Simulate fill using bid/ask/fill rules
  6. Monitor exits every bar
  7. Update portfolio and risk state
  8. Write journal row

Fill assumptions ต้องประกาศ

AssumptionConservative choice
Entrybuy spread at ask-side or mid plus slippage
Exitsell spread at bid-side or mid minus slippage
Partial fillno fill unless all legs executable
TP/SL same barassume worse sequence unless tick data proves otherwise
Stale quoteskip trade

Backtest outputs

ต้องออก trade log, equity curve, drawdown curve, cost attribution, skipped-trade log, and violation log ไม่ใช่แค่ Sharpe.

Retrieval

  1. ทำไม same-bar TP/SL ambiguity อันตราย
  2. event-driven backtest มี event อะไรบ้าง
  3. ทำไม skipped-trade log สำคัญ
