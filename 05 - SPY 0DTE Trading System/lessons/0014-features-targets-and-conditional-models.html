Features, targets, conditional models

ถ้าจะใช้ model ต้องเริ่มจาก label และ timestamp ไม่ใช่เริ่มจาก algorithm

Feature rule

Feature Engineering page ระบุว่า every feature must be available at the decision timestamp. สำหรับ 0DTE นี่หมายถึง feature ที่ใช้ตัดสิน 9:36 ต้องรู้ก่อน 9:36 จริง ๆ.

Model baseline

0DTE Trading Rules พบว่า directional classification ดีกว่า direct return prediction ในบาง setting แต่ไม่ได้แปลว่า ML ซับซ้อนยิ่งดี เริ่มจาก logistic regression / simple rules, compare against unfiltered rule, then only add complexity if OOS improves after costs.

Validation discipline

  • chronological split เท่านั้น
  • scaler/feature selector fit ใน train window เท่านั้น
  • rolling/expanding window
  • report Brier score, calibration slope, hit rate, mean net PNL, ES, drawdown

Retrieval

  1. ทำไม target engineering สำคัญกว่า model choice
  2. accuracy สูงแต่ strategy แพ้ได้อย่างไร
  3. feature leakage ใน 0DTE เกิดจากอะไร
