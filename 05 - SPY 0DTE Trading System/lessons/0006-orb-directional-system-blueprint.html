Blueprint: ระบบ ORB directional บน SPY 0DTE

สร้างตัวอย่างระบบแบบมีกรอบ แต่ยังไม่ถือว่าใช้งานจริงจนกว่าจะ validate

Core rule skeleton

  1. ก่อนตลาดเปิด: เช็ก day, VIX, macro event, liquidity expectation
  2. 9:30-9:35 ET: วัด opening range high/low
  3. หลัง 9:35: รอ decisive break
  4. เลือก direction: call spread ถ้าทะลุบน, put spread ถ้าหลุดล่าง
  5. เลือก strike: nearest OTM หรือ rule ที่ทดสอบแล้ว
  6. ส่ง limit order ตาม bid/ask discipline
  7. ออกด้วย TP/SL/time stop
  8. log ทุกเหตุการณ์

No-trade rules สำคัญกว่า entry

ถ้าระบบ edge เล็กมาก วันที่ไม่ควรเทรดมีค่าพอ ๆ กับวันที่เทรด กฎ no-trade ควรรวม macro event, spread กว้าง, data feed ผิดปกติ, option chain stale, late entry, gap bucket เสี่ยง, daily loss limit reached.

Retrieval

  1. ทำไม no-trade rule ต้องมาก่อน entry refinement
  2. trade decision object ควรมี field อะไรบ้าง
  3. ทำไม late entry ต้องเป็น violation
