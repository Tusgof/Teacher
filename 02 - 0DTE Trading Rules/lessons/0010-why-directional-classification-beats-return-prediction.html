บท 0010: ทำไมทำนายทิศทางดีกว่าทำนาย return ตรง ๆ

Table 9 คือจุดเปลี่ยนของ paper: ใน 0DTE การถามว่า "จะกำไรไหม" ทำงานดีกว่าถามว่า "จะกำไรกี่ bps"

ภาพจำ: 0DTE payoff noisy และ tail-heavy มาก การทำนายขนาด return เป๊ะ ๆ ยากเกินไป แต่การทำนายว่าผลสุทธิจะบวกหรือลบอาจพอมีสัญญาณให้ใช้

1. Return prediction คืออะไร

model แบบ return target พยายามทำนายตัวเลข เช่น strategy นี้วันนี้จะได้ +1.2 bps หรือ -0.8 bps

ปัญหาคือ 0DTE มี rare jumps และ payoff ที่ไม่เรียบ ตัวเลขจริงอาจกระโดดแรงจน model ทำนายขนาดได้ยาก

2. Direction classification คืออะไร

แทนที่จะทำนายขนาด ให้ model ตอบคำถามง่ายกว่า: net PNL วันนี้จะมากกว่า 0 ไหม

ถ้า model ให้ probability ว่าบวกมากกว่า 50% เรา long strategy ถ้าน้อยกว่า 50% เรา short strategy

3. Hard mapping vs soft mapping

Hard mapping

ถ้า p > 0.5 เข้าเต็มขนาดฝั่ง long; ถ้า p < 0.5 เข้าเต็มขนาดฝั่ง short

Soft mapping

ขนาด position ขึ้นกับความมั่นใจ เช่น p ใกล้ 0.5 ก็ถือเล็กลง

4. ทำไม logistic benchmark เหมาะ

logistic regression ให้ probability ระหว่าง 0 ถึง 1 และตรงกับคำถามว่า PNL จะบวกไหม

ในข้อมูล noisy และ sample ต่อ strategy ไม่ใหญ่มาก model ที่เรียบกว่าอาจ overfit น้อยกว่า model ซับซ้อน เช่น tree boosting
